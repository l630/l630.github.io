    一、SQL语言：结构化查询语言

    数据定义语言 DDL
数据查询语言 DQL
数据操作语言 DML
数据控制语言 DCL

    二、MySQL相关操作

    1、MySQL服务操作

    启动服务： net start mysql

关闭服务： net stop mysql

登录mysql u代表用户名 h代表ip主机 p代表密码：
mysql -u username -h host -p password

    2、数据库操作

    创建数据库          create databse [if not exists] dbname;
使用数据库          use dbname;
查询所有数据库      show databases;
删除数据库          drop database [if exists] dbname;

    3、表操作

    查看表结构         desc tbname;

查看表创建信息     show create table tbname;

创建表     create table [if not exists] tbname(
            colname    data-type [constraint] [comment],
            colname    data-typ  [constraint] [comment] );

删除表            drop table [if exists] tbname;

修改表名称        alter table old_tbname rename [to] new_tbname;

    4、列操作

    删除列            alter table tbname drop  colname;

删除外键约束       lter table tbname drop foreign key fk_name；

为表添加列：
alter table tbname add colname data-type [first | after] colname;

改变列名称：
alter table tbname change old_colname new_colname data-type [first | after] colname;

插入数据（非数字类型需要在值的前后加上单引号）：
insert into tbname(field1, field2, ..., fieldn) values (value1, value2, ..., value_n)；

    四、数据查询

    1、where查询

    查询表中所有列匹配条件的数据

    select * from tbname where condition1 [and | or] condition_n;

    2、排序

    将查询结果进行高低排序

    select* from tbname where conditions order by col_name1 asc 
select* from tbname where conditions order by col_name2 desc

    3、分组

    将查询结果进行分组（其中having 条件是为对分组后的数据进行条件筛选）

    select* from tbname where conditions 
group by col_name1 having conditions

    4、数据去重

    select distinct col_name from tbname;

    5、模糊查询

    模糊查询(%百分号, _下划线)

    select * from tbname where col_name like '%_';

    6、子查询

    子查询(子查询-该查询只能查询单列)

    select *from tbname where [只能用=、<、>、in、any、all、exisit]

    7、表链接

    select 
t1.*, t2.*,..., tn.* (多表的多列)
from tbname1 t1, tbname2 t2,..., tbname_n tn 
where conditions

    8、内链接

    select    
    t1.*,
    t2.*,
    tn.*
from    tbname1  t1
join    tbname2  t2 [on conditions]
join    tnname_n tn [on conditions]

    9、左链接

    select    
    t1.*,
    t2.*,
    tn.*
from      tbname1  t1
left join tbname2  t2 [on conditions]
left join tnname_n tn [on conditions]

    10、右链接

    select    
    t1.*,
    t2.*,
    tn.*
from       tbname1  t1
right join tbname2  t2 [on conditions]
right join tnname_n tn [on conditions]

    五、事务

    1、事务相关

    1>显示开启事务

    START TRANSACTION;

    2>显示提交事务

    COMMIT;

    3>回滚事务

     ROLLBACK;（只有当事务没有提交时 才能回滚）
（撤销事务的含义(事务中所有的操作都无效)

    4>保存事务节点 可回滚到指定节点 回滚事务节点可以有多个

    SAVEPOINT point_name;

    5>删除指定事务回滚节点

    RELEASE SAVEPOINT point_name;

    2、MySQL的事务有4个隔离级别

    READ UNCOMMITTED;    #事务会读别的事务未提交的数据 会产生脏读
READ COMMITTED;        #事务多次读取数据时，数据可能不一样（原因是别的事务修改了数据）
REPEATABLE READ;     #可重复读(MySQL默认的事务隔离级别)
SERIALIZABLE        #串行化

    3、事务的4个性质

    原子性：事务是不可分割的执行单位
隔离性：事务与事务之间相互独立
持久性：事务操作对数据库来说是永久存在的
一致性：事务操作不可破坏数据库的逻辑结构

    六、权限

    创建用户
create user 'username'@'host' [identified by 'password']

删除用户
drop user 'username'@'host'

用户授权
grant 权限1,..., 权限n on dbname.tblname to 'username'@'host'

回收用户权限
revoke 权限1,,, 权限n on dbname.tblname from 'username'@'host'

查看当前用户权限
show grants;

查看指定用户权限
show grants for 'user'@'%';

重命名用户名称
rename  user 'user'@'%' to 'newusername'@'%'

强制刷新权限
flush privileges;

    十一、数据备份

    show variables like "secure_file_priv";    该设置可限制文件导出导入

secure_file_priv为null        表示不允许导入导出

secure_file_priv        指定文件夹时，表示mysql的导入导出只能发生在指定的文件夹

secure_file_priv        没有设置时，则表示没有任何限制

    (1)MySQL核心配置文件my.ini中进行配置

    select ... into outfile 'path\filename.suffix'        查询导出 只能导出表中数据

load data infile 'path' into table tblname;        导入数据 数据格式必须和表格式匹配

mysqldump -u root -p db_name > path\filename.sql    指定数据库导出(导出数据库中所有的表)

登录并指定数据库后     source path\filename.sql

    十一、索引及优化

    （建表时）

    索引类型        索引名称          索引列明及索引长度

INDEX        [indexName]       (column(length))

UNIQUE        [indexName]       (column(length))

FULLTEXT    [indexName]       (column(length))

PRIMARY KEY (column)

    alter table tbl_name add 索引类型 index_name(column(length));

    alter table tbl_name drop index index_name;

    show index from tbl_name;

    优化思路:一定的索引设计能够增加数据查询性能, 索引并不是越多越好
一般而言, where条件后的列可设计索引, 连接关系列可设计索引
